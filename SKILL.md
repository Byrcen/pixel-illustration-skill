# 插画风格配图生成skill（像素风）

写完一篇文章后用这个 skill 自动配图：根据文章内容，在 **前30% / 中间30% / 末尾30%** 三个位置各生成一张 **像素风** 插画。

## 何时使用

当用户说类似「给这篇文章配图」「用桌面那个配图 skill」「帮文章生成插图」时使用。

## 准备工作（只需一次）

1. 复制 `config.example.json` 为 `config.json`，把 `api_key` 填入用户的 apimart key（不是占位文字）。若 `config.json` 不存在或仍是占位文字，提示用户先填好再继续。
2. 确认本机有 `python3`（脚本仅用标准库，无需 pip 安装）。

## 工作流程

### 第 1 步：拿到文章内容
- 用户给 **文件路径**（.md / .txt）→ 用 Read 读全文。
- 用户 **直接粘贴正文** → 直接用粘贴的内容。
- 两种都支持，自动判断。同时确定一个**文章名**（取标题或首句关键词，用于输出文件夹命名）。

### 第 2 步：切三段，各选一个代表画面
把全文按篇幅分成三个区域：
- **前段** = 文章前 30% 的内容
- **中段** = 中间 30%（大约 35%~65% 处）
- **末段** = 最后 30% 的内容

每段里挑一个**最有代表性、最适合视觉化的场景/意象/概念**，作为这张图要画的主体。三张图主题应彼此不同、能串起文章的叙事。

### 第 3 步：为每段写画图 prompt
每个 prompt = **该段的具体画面描述** + **下面这段固定像素风后缀（原样拼接，不要改）**：

```
像素风格 pixel art 插画，复古游戏画面质感，清晰的像素颗粒与有限色板，色彩温暖明亮，氛围亲和、有故事感和生活气息；不要写实摄影感，不要赛博朋克/霓虹/暗黑风；构图干净，主体清晰突出，留白舒适，适合内容平台配图。画面中如出现任何文字，必须使用简体中文。
```

要点：
- 画面描述要**具体**（谁/在哪/在做什么/关键物件/情绪），别只写抽象词。
- 图里若需要文字，明确写出要显示的**简体中文**字样。
- 不要在 prompt 里写真实人物姓名或受版权保护的角色。

### 第 4 步：依次生成三张图
对每一张，调用脚本（一次一张）：

```bash
python3 ~/Desktop/插画风格配图生成skill/generate.py \
  --prompt "<该段完整 prompt>" \
  --out "/Users/cry/Desktop/文章配图/<文章名>/01_前段.png" \
  --size 16:9 --resolution 2K
```

- 三张分别输出到 `01_前段.png`、`02_中段.png`、`03_末段.png`。
- 脚本会自动：提交任务 → 轮询到完成 → 立即下载（图 URL 24 小时过期，必须当场下）。
- 脚本成功时 stdout 打印保存路径，退出码 0；失败时退出码非 0 且 stderr 有原因。

### 第 5 步：汇报
告诉用户三张图的本地路径，并用 Read 把图片展示出来给用户看一眼。

## 尺寸 / 清晰度

- **默认 16:9 横向 + 2K**。
- 用户每次调用可临时指定，例如「这次用 1:1」「要 4K」→ 把 `--size` / `--resolution` 改成对应值。
- `--size` 可选：`16:9` / `1:1` / `4:3` / `3:4` / `9:16` / `auto`
- `--resolution` 可选：`1K` / `2K` / `4K`

## 输出位置

统一存到桌面：`~/Desktop/文章配图/<文章名>/`，每篇文章一个子文件夹。

## 技术参数（已写死在脚本里，一般不用动）

- 接口：`POST https://api.apimart.ai/v1/images/generations`（提交），`GET https://api.apimart.ai/v1/tasks/{task_id}`（轮询）
- 模型：`gemini-3-pro-image-preview`
- 认证：`Authorization: Bearer <config.json 里的 api_key>`
